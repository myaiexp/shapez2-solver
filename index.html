Solver Options

Options specific to the solver. This will not affect the shape explorer.

Choose the search algorithm: A* uses heuristics for faster targeted search, BFS explores breadth-first with state pruning. Constructive decomposes hard multi-quadrant targets into search-sized pieces, then assembles them by stacking.

Smaller values solve faster but may be less efficient or fail. Larger values take more time but are more likely to succeed and give better results.

Divisor for the mismatch penalty in A* heuristic. Lower values make the heuristic stricter, potentially finding simpler solutions faster. If solutions are too simple, increase this value.

If checked, all remaining shapes must match the target in order for the solution to be considered valid.

Only identify a shape as the target if its orientation is an exact match.

Only consider painting a shape if it has one layer; for cleaner-looking solutions.

Automatically remove starting shapes/colors that are not needed for the target, reducing search space.

Duplicate processing machines in the blueprint layout for higher throughput. Adds splitters and mergers automatically.

Shape Options

Maximum number of layers a shape can have. Most scenarios have a cap of 4 layers, but the Insane scenario increases the limit to 5.

Choose how you want colors to be displayed on shapes.
